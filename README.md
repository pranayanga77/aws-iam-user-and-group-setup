# AWS IAM User and Group Setup


## Objective
To create IAM users and groups for secure AWS access instead of using the root account.

## Steps Performed
1. Created IAM group
2. Attached permissions to the group
3. Created IAM user
4. Added user to group
5. Enabled console access

## AWS Best Practices Followed
- No root account usage
- Permissions assigned via groups
- Principle of least privilege

## Real-World Use Case
Used in organizations to manage employee access securely.
